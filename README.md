# Rental AI

Rental AI is a safer NYC rental search beta that helps renters avoid surprise fees and scam-risk guesswork. The public GitHub Pages homepage is now a landing page for beta signup; the previous shortlist analyzer remains available as a secondary prototype demo.

Live landing page: <https://ai-nate.github.io/rental-ai-site/>
Prototype demo: <https://ai-nate.github.io/rental-ai-site/demo.html>

Set the final Tally beta signup URL in `src/config.js` by updating `BETA_SIGNUP_TALLY_URL`. Suggested Tally fields are documented in `docs/beta-signup-tally.md`. If the value is blank, the landing page shows a clearly marked placeholder CTA and collects no renter data.

## What the prototype demo does

The secondary prototype demo starts with a renter-needs interview, builds targeted listing searches, accepts normal pasted listing text (no JSON required), and turns 3–10 found listings into a ranked shortlist of up to five places with:

- known all-in monthly cost vs. the renter's housing budget
- `within budget`, `near limit`, `over budget`, or `missing cost data` labels
- NYC-relevant fee checks for broker/no-fee claims, application fees, deposits, guarantor/income requirements, utilities, pets, parking, laundry, and move-in costs
- scam/legitimacy warnings for payment-before-tour/signing, unsafe payment channels, missing address, copied-looking marketplace text, inconsistent manager/contact info, and too-good-to-be-true rent
- commute, transit, neighborhood/borough, internet/remote-work, laundry, pets, parking, and building/manager-reputation notes
- a recommendation: `tour`, `ask first`, or `skip`
- a feedback form that records completed shortlist count, listings submitted, recommendation mix, usefulness answer, comments, optional contact, and whether the live-site CTA was clicked

This v3 build does **not** submit applications, collect SSNs, run screening, process payments, guarantee legitimacy, give legal advice about NYC housing rules, or conduct public outreach automatically. SerpApi search is GitHub Actions-only: `SERPAPI_API_KEY` belongs in the repository secret, the action pre-generates public static snippets for GitHub Pages, and the UI clearly labels them as not live per-user search. The public site always keeps one-click targeted search links and plain-text paste/analyze fallback.

## Validation workflow

Phase 0 acquisition should be owner-approved posts or direct interview asks only. After a renter completes a run:

1. Interview for neighborhood/borough, budget, layout, move timing, commute target, must-haves, and deal-breakers.
2. Use the generated search links (or a future connected search API) to find promising listings.
3. Paste normal listing snippets/pages/URLs into the no-JSON textarea and create the shortlist.
4. Ask: "Did this help you decide what to tour, ask about, or skip?"
5. Save yes/no/unsure, optional comment, and optional contact in the on-page feedback form.
6. Use **Download JSONL** and append the records to the loop's `feedback.jsonl` or the owner's chosen feedback store.
7. Judge traction only after 5 real NYC/default-city renters complete a shortlist run.

See `docs/validation-plan.md` for copy and handling notes.

## GitHub Actions search refresh

Search snippets for the GitHub Pages site are pre-generated by `.github/workflows/pages.yml` using `SERPAPI_API_KEY` from **Settings → Secrets and variables → Actions → Repository secrets**. They are static public data, not live search results for each renter.

Owner action: set the `SERPAPI_API_KEY` repository secret, then run **Actions → Build and deploy GitHub Pages → Run workflow** with `refresh_search=true`, or leave the included Monday schedule enabled. Details: `docs/github-actions-search.md`.

## Run locally

```bash
npm run dev
```

Open <http://localhost:4173>.

Static search refresh is handled by GitHub Actions, not the browser or local dev server. See `docs/github-actions-search.md` for where to set the `SERPAPI_API_KEY` repository secret and how to run or schedule the workflow.

## Validate

```bash
npm run check
```

`npm run check` executes the built-in Node test suite and creates a static production build in `dist/`.
